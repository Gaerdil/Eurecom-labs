def testModel(k= 2, n=2, fixedEnergyConstraint = True, questionD = False, nonlinearD1 = False, nonlinearD2 = False, questionE = False,increaseNbrLayer = False):
    
    # Initialization - Autoencoder (n,k)
    # M: number of messages to encode
    # k = log2(M)
    # n real-value signals (otherwise. for complex-valued signals, 2*n real numbers)
    k = k
    n = n
    M = 2**k
    R = k/n

    print(" ********* Testing autoencoder architecture ********* ")
    print(" ")
    print('M:',M,'\t','k:',k,'\t','n:',n)
    print(" ")

    # Channel parameters
    Eb_No_dB = 7
    noise = 1/(10**(Eb_No_dB/10))
    noise_sigma = np.sqrt(noise)
    beta = 1/(2*R*(10**(Eb_No_dB/10)))
    beta_sqrt = np.sqrt(beta)


    # Creating a training set and test set 
    eye_matrix = np.eye(M)
    x_train = np.tile(eye_matrix, (1000, 1))  
    x_test = np.tile(eye_matrix, (100, 1)) 
    rd.shuffle(x_train)
    rd.shuffle(x_test)


    # Printing the shape of x_train and x_test
    print(x_train.shape)  
    print(x_test.shape) 


    #Bit Error Rate
    def BER(y_true, y_pred):
        return K.mean(K.not_equal(y_true, K.round(y_pred)), axis=-1)  


    # Autoencoder definition/construction

    # Take the input and convert it to Keras tensors
    input_signal = Input(shape=(M,))
    encoded_input = Input(shape=(n,))

    #Transmitter Neural Network (encoder)


    ######## YOUR CODE STARTS HERE #######
    #### Create two dense layers, with input 'input_signal' and output 'encoded' ####
    ### The first layer has activation ReLU and the second linear
    if  questionE:   #We are at any other question than question D
        print("--- Question e) ---")
        if increaseNbrLayer:
            layer_T1 = Dense(M, input_shape=(M,), activation = K.relu)(input_signal)
            layer_T2 = Dense(M,  activation = K.relu)(layer_T1)
            encoded = Dense(n, activation = K.relu) (layer_T2 )
        else:
            encoded = Dense(n, input_shape=(M,))(input_signal)
            
    elif questionD:
        print("--- Question d) ---")
        if nonlinearD1: #We add some nonlinearity
            print("Test with more nonlinearity -input")
            layer_T = Dense(M, input_shape=(M,), activation = K.relu)(input_signal)
            encoded = Dense(n, activation = K.relu) (layer_T )
        else: #we test the linear dense layers
            print("Test with no nonlinearity -input")
            layer_T = Dense(M, input_shape=(M,))(input_signal)
            encoded = Dense(n) (layer_T )
    else:
        layer_T = Dense(M, input_shape=(M,), activation = K.relu)(input_signal)
        encoded = Dense(n) (layer_T )
    ######## END OF YOUR CODE      #######



    # If you want to use regularizers or perform batch normalization, uncomment the below 
    #encoded = ActivityRegularization(l2=0.02)(encoded)
    #encoded = BatchNormalization()(encoded)

    # Signal constraints
    if fixedEnergyConstraint:
        print("Fixed energy constraint")
        encoded = Lambda(lambda x: np.sqrt(n) * K.l2_normalize(x, axis=1))(encoded) #energy constraint
    ######## YOUR CODE STARTS HERE #######
    #### The above command implement a fixed energy constraint. For the question on implementing an average power constraint ####
    #### Comment the above line and write your own command for the power constraint ####
    elif not fixedEnergyConstraint:
        print("Average energy constraint")
        ## Not sure about this
        #encoded = BatchNormalization(momentum = 0)(encoded)
        def average_energy_constraint(x):
            return tf.cond(tf.norm(x, ord=2) > 1, lambda: K.l2_normalize(x, axis=1), lambda: x)
        encoded = Lambda(average_energy_constraint)(encoded)        
    ######## END OF YOUR CODE      #######






    # Channel Layer
    encoded_noise = GaussianNoise(beta_sqrt)(encoded)

    # Receiver Neural Network (decoder)

    ######## YOUR CODE STARTS HERE #######
    #### Create two dense layers, with input 'encoded_noise' and output 'decoded' ####
    ### The first layer has activation ReLU and the second softmax
    if questionE:   #We are at any other question than question D
        if increaseNbrLayer: #keep the same
            layer_R = Dense(M, activation = K.relu)(encoded_noise)
            decoded = Dense(M, activation = K.softmax)(layer_R)
            
        else:
            decoded = Dense(M, activation = K.softmax)(encoded_noise)
            
       
    elif questionD:
        if nonlinearD2: #We add some nonlinearity with the softmax
            print("Test with less nonlinearity in question d -output")
            layer_R = Dense(M)(encoded_noise)
            decoded = Dense(M, activation = K.softmax)(layer_R)

    autoencoder = Model(inputs=input_signal, outputs=decoded)

    encoder = Model(inputs=input_signal, outputs=encoded)

    autoencoder.summary()

    plt.xlabel('Epoch')


    # making decoder from full autoencoder
    deco = autoencoder.layers[-2](encoded_input)
    deco = autoencoder.layers[-1](deco)
    decoder = Model(encoded_input, deco)


    # Calculating BER from -2dB to 10dB Eb/No

    # Generating data of size N
    N=100000
    test_label = np.random.randint(M,size=N)

        temp=np.zeros(M)

        EbNo=10.0**(EbNodB_range[n_i]/10.0)

        noise_mean = 0
        no_errors = 0
        nn = N
        noise = noise_std*np.random.randn(nn,n)
        encoded_signal = encoder.predict(test_data) 
        final_signal = encoded_signal+noise
        pred_final_signal =  decoder.predict(final_signal)
        pred_output = np.argmax(pred_final_signal,axis=1)
        no_errors = (pred_output != test_label)
        no_errors =  no_errors.astype(int).sum()
        ber[n_i] = no_errors / nn 
        print ('Eb/No:',EbNodB_range[n_i],'BLER:',ber[n_i])


    # ploting BER curve

COMMENTS¶

For n=2, k=2¶

The constellation looks a like Q-PSK. Indeed the four points are around a circle. QPSK uses four points on the constellation diagram, equispaced around a circle. This constellation is like a rotation of the Q-PSK with little changes, the four symbols are not equispaced. Since the model is defined with a fixed power constraint, it forces the symbols to lie on the circle with a radius of root 2, and to maximize the BLER, symbols should not be too close to each other.

For n=2, k=4¶

It is almost like 16-PSK with a very small rotation. The more symbols there are, the harder it is to differentiate between them since there are more symbols closer than before because we are using the same fixed energy constraint. So the model is trying to learn a constellation like 16-PSK, since this one offers the best minimum distance between two symbols.

Comments¶

Results are very different than what was expected. Keeping the softmax output seems mandatory (to get a probability as an output). However, surprisingly, it seems that keeping some linear layers (with linear activation) is helpfull. Even if we expected at first glance non linear functions such as relu to help achieving better results, we can think of a few arguments that could explain why linear activations can help achieve better results. To begin with, the application is a communication system, where coding/modulation processes are often linear. Therefore, if we have a linear coding process, the decoding process could also be linear (we could find some kind of symetry between the two processes). So, it means that if we choose linear activations in the encoder part our autoencoder, it might be better to also choose linear activations for the decoder part.

Also, in our specific case, we want the output to be similar to the input. Therefore, a linear activation could be convenient, much more than some other types of activations, as it will produce outputs that are similar to the inputs.

It seems that ReLU is a bad option in the specific case of dimensionnality reduction. Maybe because we lose too much information by setting negative inputs to 0? Linear activations are seemingly far better activation functions than relu functions, from our results. Indeed, We have understood that, like PCA, a linear activation does some kind of selection regarding features selecting the the most significant ones. And that, there is equivalence if we had one linear layer in our encoder. However, it does not mean at all that the best type of activation function in our autoencoder setting is necessarly the linear one (in this very specific setting, linear activation is not as good as a certain rarely used cosine activation function).

Also, linear layers are also good in our setting, because of the simplicity of our model/channels/noise model. With other types of models, more complex, with multiplicative noise for instance, the linear layers might not be that great, and ReLU might be a better option
